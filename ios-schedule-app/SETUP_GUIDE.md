# iOSスケジュールアプリ - 動作確認手順

このガイドでは、作成したスケジュールアプリをXcodeで実行して動作確認する手順を説明します。

## 必要な環境

- macOS 12.0以降
- Xcode 13.0以降
- iOS 15.0以降のシミュレータまたは実機

## 手順1: Xcodeプロジェクトの作成

### 1-1. Xcodeを起動

Xcodeを起動して、「Create a new Xcode project」を選択します。

### 1-2. テンプレートの選択

1. 上部のタブで **「iOS」** を選択
2. **「App」** を選択
3. 「Next」をクリック

### 1-3. プロジェクト設定

以下の設定を入力します：

- **Product Name**: `ScheduleApp`
- **Team**: (個人の場合は「None」でOK)
- **Organization Identifier**: `com.yourname` (任意)
- **Interface**: **SwiftUI** を選択 ⚠️重要
- **Language**: **Swift** を選択
- **Storage**: Core Dataのチェックを外す
- **Include Tests**: チェックを外してOK

「Next」をクリックして、保存場所を選択します。

### 1-4. デフォルトファイルの削除

プロジェクトが作成されたら、左側のナビゲータで以下のファイルを削除します：

- `ContentView.swift` (既存のもの)

ゴミ箱アイコンをクリックして「Move to Trash」を選択します。

## 手順2: ソースファイルの追加

### 2-1. Finderでファイルを確認

Finderで以下のパスを開きます：

```
/Users/motonishikoudai/verantyx_v6/ios-schedule-app/ScheduleApp/
```

以下の6つのSwiftファイルがあることを確認：

- ✅ Schedule.swift
- ✅ ScheduleViewModel.swift
- ✅ ContentView.swift
- ✅ AddScheduleView.swift
- ✅ ScheduleDetailView.swift
- ✅ ScheduleAppApp.swift

### 2-2. Xcodeにファイルを追加

1. Xcodeの左側ナビゲータで **「ScheduleApp」** フォルダを右クリック
2. **「Add Files to "ScheduleApp"」** を選択
3. Finderで上記のSwiftファイルを**すべて選択**
4. 以下のオプションを確認：
   - ✅ **「Copy items if needed」** にチェック
   - ✅ **「Create groups」** を選択
   - ✅ **「Add to targets: ScheduleApp」** にチェック
5. 「Add」をクリック

### 2-3. ファイル構成の確認

左側のナビゲータで以下のような構成になっているか確認：

```
ScheduleApp/
├── ScheduleAppApp.swift
├── Schedule.swift
├── ScheduleViewModel.swift
├── ContentView.swift
├── AddScheduleView.swift
├── ScheduleDetailView.swift
└── Assets.xcassets
```

### 2-4. デフォルトのApp構造体の削除

`ScheduleAppApp.swift` が重複している場合（プロジェクト作成時に自動生成されたものと追加したもの）、古い方を削除します。

残すのは、以下の内容のファイルです：

```swift
@main
struct ScheduleAppApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

## 手順3: ビルドと実行

### 3-1. シミュレータの選択

Xcodeの上部ツールバーで、シミュレータを選択します：

- おすすめ: **iPhone 14** または **iPhone 15**
- 「Product」→「Destination」→「iPhone 14」を選択

### 3-2. ビルド

1. **⌘ + B** (Command + B) を押してビルド
2. エラーがないことを確認

### 3-3. 実行

1. **⌘ + R** (Command + R) を押して実行
2. シミュレータが起動してアプリが表示されます

## 手順4: 動作確認

### 4-1. 初期画面の確認

アプリが起動したら、以下が表示されることを確認：

✅ 上部にカレンダーが表示される
✅ 検索バーが表示される
✅ サンプルのスケジュールが3件表示される
✅ 右上に「+」ボタンがある

### 4-2. カレンダー操作の確認

1. **月の切り替え**
   - カレンダー上部の「<」「>」ボタンをタップ
   - 前月・次月に移動することを確認

2. **日付選択**
   - カレンダーの日付をタップ
   - 選択した日付が青い背景で表示される
   - 下部のスケジュール一覧が選択日のものに変わる

### 4-3. スケジュール追加の確認

1. 右上の **「+」ボタン** をタップ
2. 新規スケジュール画面が表示される
3. 以下を入力：
   - **タイトル**: 「テストスケジュール」
   - **説明**: 「動作確認用」
   - **カテゴリ**: 「テスト」
   - **場所**: 「自宅」
4. 日付と時刻を設定
5. 優先度を選択（高・中・低）
6. **「保存」** をタップ
7. メイン画面に戻り、追加したスケジュールが表示される

### 4-4. スケジュール詳細の確認

1. リストからスケジュールをタップ
2. 詳細画面が表示される
3. 以下が表示されることを確認：
   - タイトル
   - 優先度（色付き）
   - 日付
   - 時間
   - カテゴリ
   - 場所
   - 説明

### 4-5. スケジュール編集の確認

1. 詳細画面で **「編集」** をタップ
2. 編集モードに切り替わる
3. タイトルを変更
4. **「保存」** をタップ
5. 変更が反映される

### 4-6. 完了チェックの確認

1. リストの左側にある **○アイコン** をタップ
2. ✓マークが表示される
3. スケジュールが半透明になり、取り消し線が引かれる
4. もう一度タップすると未完了に戻る

### 4-7. 検索機能の確認

1. 検索バーに「テスト」と入力
2. 該当するスケジュールのみが表示される
3. 検索文字列を削除すると全スケジュールが表示される

### 4-8. 削除機能の確認

**方法1: スワイプで削除**
1. リストのスケジュールを左にスワイプ
2. 「Delete」ボタンが表示される
3. タップすると削除される

**方法2: 詳細画面から削除**
1. スケジュールをタップして詳細画面を開く
2. 下部の **「削除」ボタン** をタップ
3. 確認ダイアログが表示される
4. **「削除」** をタップ
5. メイン画面に戻り、スケジュールが削除される

### 4-9. データ永続化の確認

1. スケジュールをいくつか追加・変更
2. アプリを **完全に終了**（シミュレータのホームボタンをダブルクリック→上にスワイプ）
3. アプリを再度起動
4. 変更したデータが保持されていることを確認

## トラブルシューティング

### ビルドエラーが出る場合

**エラー: "Cannot find 'ContentView' in scope"**

→ `ScheduleAppApp.swift` が正しく設定されているか確認してください。

**エラー: Duplicate symbol**

→ ファイルが重複していないか確認してください。特に `ScheduleAppApp.swift` と `ContentView.swift` が2つずつないか確認。

### アプリが起動しない場合

1. シミュレータを再起動
   - 「Simulator」メニュー → 「Quit Simulator」
   - Xcodeから再度実行

2. ビルドフォルダをクリーン
   - 「Product」→「Clean Build Folder」（⇧⌘K）
   - 再度ビルド

### レイアウトが崩れる場合

- iPhoneのシミュレータを使用しているか確認
- iPadではレイアウトが異なる場合があります

## 実機での動作確認

### 実機で動かす場合

1. iPhoneをUSBケーブルでMacに接続
2. Xcodeの上部ツールバーで接続したiPhoneを選択
3. 「Signing & Capabilities」タブで自分のApple IDを設定
4. ⌘ + R で実行

**注意**: 初回は「信頼されていない開発者」エラーが出る場合があります。
→ iPhone の「設定」→「一般」→「VPNとデバイス管理」から証明書を信頼してください。

## 次のステップ

動作確認ができたら、以下のカスタマイズを試してみてください：

- サンプルデータの内容を変更
- 優先度の色を変更
- 新しい機能を追加（通知、リマインダーなど）
