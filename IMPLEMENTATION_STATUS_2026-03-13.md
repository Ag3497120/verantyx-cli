## 技術的洞察

### なぜ.jcrossなのか

#### 1. 日本語の述語中心性
```jcross
// 日本語: 述語（状態）が中心
パターン.位置 $= 理想位置  // 「である」が中心

// 英語: 主語が中心
pattern.position = ideal_position  // patternが中心
```

宣言型プログラミングは「状態の宣言」であり、述語中心の日本語と親和性が高い。

#### 2. 高コンテキスト性
```jcross
// 文脈から明らか → 主語省略可能
もし スコア >= 閾値 {
    信頼度 = "高"  // 「これの」信頼度（文脈依存）
}
```

Cross空間の「空間的文脈」と日本語の「言語的文脈」が対応。

#### 3. 並列概念の自然な表現
```jcross
// 各要素が同時に存在
空間 パターン群 {
    ギットハブ $= {...}
    タスク $= {...}
    質問 $= {...}
}

// 空間変換（全要素が同時）
共鳴群 $= パターン群 ~> 空間 {...}
```

### Kofdai vs Neumann

| 観点 | Neumann型 | Kofdai型 |
|------|-----------|----------|
| データ更新 | 上書き（破壊） | 再配置（保存） |
| パターンマッチング | if/elseの連鎖 | 並列共鳴 |
| 選択ロジック | 明示的分岐 | 自然選択（最大値） |
| 実行モデル | 順次処理 | 並列概念 |
| 言語 | 英語・命令型 | 日本語・宣言型 |
