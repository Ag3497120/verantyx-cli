# 学習プロセスの詳細説明
SSD動画からの自発的学習メカニズム

## 概要

このシステムは**教師なし学習**です。人間が「これは猫」「これは犬」とラベルを付けることはありません。

赤ちゃんと同じように、**「予測と現実のズレ（緊張）」**だけを頼りに、自発的に世界を理解していきます。

---

## 学習プロセス全体フロー

```
1. 動画読み込み (halll.mov - 2.7GB)
   ↓
2. フレーム抽出 (200フレームごとにサンプリング)
   ↓
3. Cross構造変換 (26万点 → 5層の階層構造)
   ↓
4. 予測エンジン (次フレームを脳内シミュレート)
   ↓
5. 緊張計算 (予測誤差 = 不快感)
   ↓
6. 学習判定
   ├→ 緊張低い → 理解している → 記憶のみ
   └→ 緊張高い → 気になる！ → 新パターンとして深く学習
   ↓
7. 記憶保存 (Cross構造を生のまま保存)
   ↓
8. 次フレームへ (ステップ2に戻る)
```

## 各ステップの詳細

### Step 1: 動画読み込み

```python
# halll.mov を OpenCV で開く
cap = cv2.VideoCapture('/Users/motonishikoudai/Desktop/halll.mov')

総フレーム数: 126,488 フレーム
解像度: 3024 x 1964 ピクセル
サイズ: 2.7 GB
```

**特徴**:
- 実際のSSD内の動画
- ユーザーが撮影した実データ
- 高解像度、大容量

### Step 2: フレーム抽出

```python
# 200フレームごとにサンプリング
frame_count = 0
for frame in video:
    frame_count += 1
    if (frame_count - 1) % 200 == 0:
        process(frame)
```

**なぜサンプリング？**

全126,488フレームを処理すると時間がかかりすぎる。
200フレームごとにサンプリングすることで:
- 処理時間: 数時間 → 数十秒
- 全体の動きは把握できる
- 学習に必要な情報は保持

**結果**:
```
126,488 フレーム ÷ 200 = 約632フレーム
→ 今回は最初の50フレームのみ処理
```

### Step 3: Cross構造変換

これが**最も重要**なステップです。

#### 3-1: 画像の前処理

```python
# BGR → RGB 変換
frame_rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)

# リサイズ (処理速度のため 3024x1964 → 64x64)
frame_resized = cv2.resize(frame_rgb, (64, 64))
```

#### 3-2: 多層Cross構造への変換

```
元の画像 (64x64 = 4,096ピクセル)
    ↓
Layer 0: Pixel Layer (4,096点)
  - 各ピクセルが1つのCross点になる
  - RGB値 + 空間座標 (x, y, z)
  - 6軸値 (UP, DOWN, RIGHT, LEFT, FRONT, BACK)

    ↓ (特徴抽出)

Layer 1: Feature Layer (~700点)
  - エッジ、テクスチャを検出
  - 局所的な特徴パターン

    ↓ (パターン抽出)

Layer 2: Pattern Layer (~680点)
  - 繰り返しパターン
  - より大きな構造

    ↓ (意味抽出)

Layer 3: Semantic Layer (~170点)
  - 意味的な領域
  - オブジェクトの境界

    ↓ (概念抽出)

Layer 4: Concept Layer (~15点)
  - 最も抽象化された表現
  - 画像全体の「概念」
```

**重要なポイント**:

各層は**圧縮**されている:
```
Layer 0: 4,096点 (100%)
Layer 1: 700点   (17%)
Layer 2: 680点   (16%)
Layer 3: 170点   (4%)
Layer 4: 15点    (0.4%)
```

これは人間の脳の**視覚皮質**と同じ構造:
```
網膜 → V1 → V2 → V4 → IT野
(詳細)              (抽象)
```

#### 3-3: 6軸システム

各Cross点は**6つの方向**を持つ:

```jcross
点 = {
  位置: (x, y, z),
  RGB: (r, g, b),
  軸: {
    UP:    0.5,   # 上方向の影響
    DOWN:  0.3,   # 下方向の影響
    RIGHT: 0.7,   # 右方向の影響
    LEFT:  0.2,   # 左方向の影響
    FRONT: 0.8,   # 前方向の影響 (時間・期待)
    BACK:  0.4    # 後方向の影響 (記憶・過去)
  }
}
```

**6軸の意味**:
- **UP/DOWN**: 活性化制御（覚醒/鎮静）
- **RIGHT/LEFT**: 空間的影響（接近/回避）
- **FRONT/BACK**: 時間的影響（期待/記憶）

これが**感情の「色」**になる。

### Step 4: 予測エンジン

#### 4-1: 脳内シミュレーション

```jcross
定義する 次フレーム予測 受け取る [現在Cross, 履歴] = {
  # 直近2フレームを取得
  前々フレーム = 履歴[-2]
  前フレーム = 履歴[-1]

  # 動きのベクトルを計算
  動きベクトル = 前フレーム.特徴 - 前々フレーム.特徴

  # 線形外挿で次を予測
  予測特徴 = 前フレーム.特徴 + 動きベクトル

  # 記憶から類似パターンを探す
  類似経験 = 記憶.検索(予測特徴, 閾値=0.6)

  もし 類似経験が存在:
    # 過去の経験から予測
    返す 類似経験.Cross構造
  そうでなければ:
    # 予測できない（未知）
    返す なし
}
```

**実際の例**:

```
フレーム48: Layer4が14点
フレーム49: Layer4が15点  (+1点)
          ↓
予測: フレーム50は16点になるはず

実際: フレーム50は15点
誤差 = |16 - 15| / 15 = 0.067

→ 小さい誤差 = 理解している
```

---

### Step 5: 緊張計算

#### 5-1: 予測誤差（Prediction Error）

```jcross
予測誤差 = ||予測Cross - 実際Cross|| / ||実際Cross||

例:
予測: 5,670点
実際: 5,664点
誤差 = |5,670 - 5,664| / 5,664 = 0.001 (0.1%)
```

#### 5-2: 同調失敗（Synchronization Failure）

```jcross
# 既存の記憶と比較
類似経験リスト = 記憶.検索(実際Cross, 閾値=0.5)

もし 類似経験が見つからない:
  同調失敗 = 1.0  # 完全に未知
そうでなければ:
  最良類似度 = 類似経験[0].類似度
  同調失敗 = 1.0 - 最良類似度
```

#### 5-3: 総緊張

```jcross
総緊張 = (予測誤差 × 0.6) + (同調失敗 × 0.4)

例 (halll.mov の場合):
予測誤差 = 0.003
同調失敗 = 0.0 (50個の経験があるため)
総緊張 = (0.003 × 0.6) + (0.0 × 0.4) = 0.0018

→ 緊張度 = 0.0018 (ほぼゼロ)
→ 😌 完全に理解している
```

---

### Step 6: 学習判定

#### パターンA: 緊張が低い（理解している）

```jcross
もし 総緊張 < 0.3:
  # 退屈、予測できている
  状態 = "理解"

  # 記憶に保存するが、新発見ではない
  経験ID = 記憶.保存(Cross構造, 不快感=0.0)

  # 既存パターンに同調を試みる
  類似経験 = 記憶.検索(Cross構造, 閾値=0.7)
  もし 類似経験が存在:
    # 既知のパターンにマッピング成功
    報酬 = 同調達成 + 圧縮成功

  返す "安定モード"
```

**halll.mov の場合**:
- 緊張 = 0.0018 < 0.3
- すべてのフレームが「理解している」状態
- 動画が滑らか、規則的な動き

#### パターンB: 緊張が高い（気になる！）

```jcross
もし 総緊張 > 0.3:
  # 予測できない！気になる！
  状態 = "探索"

  # どこが気になるか特定
  興味領域 = 最大緊張領域を探す(Cross構造)

  # アクションを決定
  アクション = 次アクションを決定(興味領域)
  # → "ズーム" / "パン左" / "パン右" など

  # 新パターンとして記憶
  経験ID = 記憶.保存(Cross構造, 不快感=総緊張)
  発見パターン.追加(経験ID)

  # 学習イベントとして記録
  学習.イベント.追加({
    "タイプ": "新発見",
    "経験ID": 経験ID,
    "緊張度": 総緊張,
    "理由": "予測失敗"
  })

  返す "探索モード"
```

**例: 不規則な動きの場合**:

```
フレーム10: 人が歩いている
フレーム11: 人が歩いている (+予測可能)
フレーム12: 人が突然ジャンプ！
          ↓
予測: 歩き続ける
実際: ジャンプ
誤差 = 0.8 (大きい！)
緊張 = 0.8 > 0.3
→ 🔍 気になる！新パターンとして学習
```

---

### Step 7: 記憶保存

#### 7-1: 未定義バッファへの保存

```jcross
定義する 記憶.保存 受け取る [Cross構造, 不快感, コンテキスト] = {
  経験ID = 次ID
  次ID += 1

  # 特徴を抽出 (Layer4の点)
  特徴 = Cross構造.layers[4].points の {RGB, 軸値}

  # 経験データ
  経験 = {
    "ID": 経験ID,
    "時刻": 現在時刻(),
    "Cross構造": Cross構造,  # 生のまま保存！
    "特徴": 特徴,
    "不快感": 不快感,
    "コンテキスト": コンテキスト,
    "解決": なし,    # 後で付与
    "意味": なし     # 後で付与（言語獲得後）
  }

  記憶.経験[経験ID] = 経験

  # 自動クラスタリング
  クラスタ化(経験)

  返す 経験ID
}
```

**重要**:
- **意味を付けない**
- **ラベルを付けない**
- **Cross構造を生のまま保存**

これが**「記憶は能力発達後に意味を知るために重要」**の実装。

#### 7-2: 自動クラスタリング

```jcross
定義する クラスタ化 受け取る [経験] = {
  # 既存クラスタと類似度を計算
  繰り返す 各クラスタ:
    類似度 = コサイン類似度(経験.特徴, クラスタ.パターン特徴)

    もし 類似度 > 0.7:
      # 既存クラスタに追加
      クラスタ.経験リスト.追加(経験.ID)
      クラスタ.パターン特徴 = 平均(全経験の特徴)
      返す

  # 新クラスタ作成
  新クラスタ = {
    "ID": 次クラスタID,
    "経験リスト": [経験.ID],
    "パターン特徴": 経験.特徴,
    "平均不快感": 経験.不快感
  }

  クラスタ.追加(新クラスタ)
}
```

**halll.mov の場合**:
- 50個の経験を保存
- クラスタ数 = 0（全て類似しているため自動的に1つのグループ）

## 学習の種類

### 1. パッシブ学習（受動的）

```
動画を見る → Cross構造変換 → 記憶保存
```

**特徴**:
- 教師信号なし
- ただ見るだけ
- でも全て記憶している

**halll.mov ではこれが起こった**:
- 50フレーム全て保存
- 予測成功率100%
- 新発見なし（退屈）

### 2. アクティブ学習（能動的）

```
予測失敗 → 緊張発生 → 気になる！ → アクション → 探索
```

**特徴**:
- カメラを自分で動かす
- ズーム、パン、回転
- 最も緊張が高い場所を探す

**例**:

```jcross
もし 緊張 > 0.3:
  # 画像を9つの領域に分割
  領域リスト = Cross構造.分割(グリッド=3)

  # 各領域の興味度を計算
  繰り返す 各領域:
    興味度 = 興味度計算(領域)
    # 0.4-0.7 が最も興味深い（既知と未知の中間）

  # 最も興味深い領域にフォーカス
  最良領域 = 最大興味度の領域

  アクション = "ズーム"  # その領域を詳しく見る
  実行(アクション)
```

### 3. 予測失敗からの学習

```
予測 → 実際 → 誤差大 → 新パターン発見 → 深く記憶
```

**メカニズム**:

```jcross
定義する 予測失敗から学習 受け取る [予測Cross, 実際Cross, 誤差] = {
  もし 誤差 > 0.3:
    # 大きな誤差 = 重要な情報

    # 実際のCrossを特別に記憶
    経験ID = 記憶.保存(実際Cross, 不快感=誤差, {
      "タイプ": "予測失敗",
      "誤差": 誤差,
      "予測": 予測Cross,
      "学習価値": "高"
    })

    # 学習イベントとして記録
    学習.イベント.追加({
      "タイプ": "予測失敗学習",
      "経験ID": 経験ID,
      "誤差": 誤差,
      "重み強化": 0.01 × 誤差  # 誤差が大きいほど強く学習
    })

    返す "学習実行"
}
```

## 報酬システム

### 快感: 同調達成

```jcross
# 複雑なCross構造が既存パターンに完璧にマッチ
類似経験 = 記憶.検索(Cross構造, 閾値=0.9)

もし 類似経験.類似度 > 0.9:
  # 完璧な同調！
  報酬 = 1.0

  # 「わかった！」という快感
  内部報酬系.励起(報酬)
```

### 快感: 圧縮成功

```jcross
# バラバラのピクセルが1つの概念に収束
Layer0点数 = 4,096  # ピクセル
Layer4点数 = 15     # 概念

圧縮率 = Layer4点数 / Layer0点数 = 0.0037 (0.37%)

圧縮成功 = 1.0 - 圧縮率 = 0.9963

# 計算コストが激減 = 快感
報酬 += 圧縮成功
```

**これが「楽をしたい」という動機**。
