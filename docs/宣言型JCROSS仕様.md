# .jcross 宣言型仕様 - 完全日本語版

## 概要

.jcrossは**完全日本語プログラミング言語**です。
従来の英語ベースの命令型言語とは異なり、日本語の述語中心性を活かした宣言型パラダイムを実現します。

## 言語哲学

### なぜ日本語なのか

1. **述語中心の論理**
   - 英語: Subject-Verb-Object（主語が中心）
   - 日本語: Subject-Object-Predicate（述語が中心）
   - プログラムの「意図」は述語に込められる

2. **高コンテキスト性**
   - 日本語は文脈依存度が高い
   - Cross空間の「空間的文脈」と親和性が高い
   - 省略可能な主語 → 暗黙的な依存関係

3. **柔軟な語順**
   - 日本語の語順の自由度 → 等価関係の自然な表現
   - 「AはBである」=「BであるのはA」

## 基本構文

### Stage 2: 手続き型（現在）

```jcross
// 関数定義
関数 計算(値) {
    結果 = 値 * 2
    返す 結果
}

// 繰り返し
各要素 IN リスト {
    表示(各要素)
}

// 条件分岐
もし 条件 {
    動作A
} そうでなければ {
    動作B
}
```

### Stage 3: 宣言型（目標）

```jcross
// 等価宣言
結果 $= 値 * 2

// 空間宣言
空間 パターン群 {
    ギットハブ $= {キーワード群: ["feat:", "fix:"]}
    タスク $= {キーワード群: ["TODO", "タスク"]}
}

// 空間変換
共鳴群 $= パターン群 ~> 空間 {
    各パターン ~> 共鳴 WHERE {
        共鳴.スコア $= 内積(入力波, パターン.キーワード群)
    }
}

// 制約付き検索
最良 $= 見つける(共鳴群) WHERE {
    最良.スコア $= 最大(共鳴群.スコア)
}
```

## 新しい演算子

### 1. `$=` 等価宣言演算子

**意味**: 「〜は〜と等価である」

```jcross
// 従来（代入）
x = 1 + 2  // xに3を代入（時間的）

// 宣言型（等価）
x $= 1 + 2  // xと3は等価である（空間的）
```

**違い**:
- `=` : 左辺に右辺を代入（計算の順序がある）
- `$=` : 左辺と右辺が等価（順序がない、同時存在）

### 2. `空間` 空間宣言

**意味**: 「全要素が同時に存在する空間」

```jcross
空間 パターン群 {
    // 全て同時に存在
    ギットハブ $= {名前: "github"}
    タスク $= {名前: "todo"}
    質問 $= {名前: "question"}
}
```

**特徴**:
- 順序がない（どれが先でもない）
- 全要素が並列に存在
- FOR loopで順次処理しない

### 3. `~>` 空間変換演算子

**意味**: 「空間から空間への写像」

```jcross
// パターン空間 → 共鳴空間
共鳴群 $= パターン群 ~> 空間 {
    各パターン ~> 共鳴 WHERE {
        // 変換規則
    }
}
```

**概念**:
- 元空間の全要素が同時に変換される
- FOR loopではなく、写像関数
- 行列演算と対応

### 4. `WHERE` 制約宣言

**意味**: 「制約条件」

```jcross
最良 $= 見つける(共鳴群) WHERE {
    最良.スコア $= 最大(共鳴群.スコア)
    最良.スコア >= 閾値
}
```

**違い**:
- `もし`: 条件分岐（順次処理）
- `WHERE`: 制約充足（並列解決）

## 完全な例: 全同調システム

```jcross
// ========================================
// パターン空間の宣言
// ========================================

空間 パターン群 {
    ギットハブ $= {
        キーワード群: ["feat:", "fix:", "docs:"],
        閾値: 0.7,
        動作: "ギットハブワークフロー起動"
    }

    タスク $= {
        キーワード群: ["TODO", "タスク", "予定"],
        閾値: 0.75,
        動作: "スケジューラ起動"
    }

    質問 $= {
        キーワード群: ["とは", "って何", "どうやって"],
        閾値: 0.6,
        動作: "質問応答システム起動"
    }
}

// ========================================
// 入力エネルギー波
// ========================================

入力波 $= {
    文: "feat: 日本語宣言型言語の実装",
    エネルギー: 正規化(文)
}

// ========================================
// 全同調（全パターンが同時に共鳴）
// ========================================

共鳴群 $= パターン群 ~> 空間 {
    各パターン ~> 共鳴 WHERE {
        // 共鳴度 = 内積（エネルギー波 × パターン）
        共鳴.スコア $= 内積(
            入力波.エネルギー,
            パターン.キーワード署名
        )

        // 信頼度判定（制約）
        共鳴.信頼度 $= "高" IF 共鳴.スコア >= パターン.閾値
        共鳴.信頼度 $= "中" IF (共鳴.スコア >= 0.5 AND 共鳴.スコア < パターン.閾値)
        共鳴.信頼度 $= "低" IF 共鳴.スコア < 0.5
    }
}

// ========================================
// Logic Resolution（最大共鳴の自然選択）
// ========================================

最良共鳴 $= 見つける(共鳴群) WHERE {
    最良共鳴.スコア $= 最大(共鳴群, ラムダ 共鳴 { 共鳴.スコア })
}

// ========================================
// Cross空間での再配置
// ========================================

CROSS 共鳴空間 {
    // 6次元座標系
    次元群 $= [
        前後: "品質（成功率）",
        上下: "使用頻度",
        左右: "新しさ"
    ]

    // 各パターンの新位置を等価関係で宣言
    各パターン IN パターン群 ~> 新位置 WHERE {
        // 成功率に基づくFRONT/BACK
        前後 $= パターン.成功数 / パターン.使用数

        // 使用頻度に基づくUP/DOWN
        上下 $= 最小(パターン.使用数 / 100.0, 1.0)

        // 新しさに基づくLEFT/RIGHT
        左右 $= 最大(1.0 - (経過日数 / 365.0), 0.0)

        // エネルギー最小化（物理法則）
        位置エネルギー $= 距離二乗(新位置, 理想位置)
        位置エネルギー $= 最小  // 最小エネルギー状態へ収束
    }
}

// ========================================
// 出力
// ========================================

表示("🌊 入力波: " + 入力波.文)
表示("🎯 最良共鳴: " + 最良共鳴.パターン名)
表示("   スコア: " + 文字列(最良共鳴.スコア * 100) + "%")
表示("   信頼度: " + 最良共鳴.信頼度)
```

## Stage 2からStage 3への移行

### Stage 2（手続き型エミュレーション）

現在の実装では、宣言型構文をStage 2の手続き型で**エミュレート**しています：

```jcross
// Stage 3の宣言型構文（理想）
共鳴群 $= パターン群 ~> 空間 { ... }

// ↓ Stage 2でエミュレート（実際の動作）
関数 空間写像(要素群, 変換関数) {
    結果 = []
    各要素 IN 要素群 {  // まだFOR loopを使用
        変換後 = 変換関数(各要素)
        結果 = 結果 + [変換後]
    }
    返す 結果
}

共鳴群 = 空間写像(パターン群, 変換関数)
```

### Stage 3（真の宣言型）

最終目標では、FOR loopを完全に排除：

```jcross
// Neural Engineが直接実行
共鳴群 $= パターン群 ~> 空間 {
    各パターン ~> 共鳴 WHERE {
        共鳴.スコア $= 内積(入力波, パターン)
    }
}

// ↓ NumPy行列演算に変換
// pattern_matrix @ input_vector
```
