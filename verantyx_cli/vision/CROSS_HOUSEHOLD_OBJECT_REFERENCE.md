# Cross構造による家庭内物体20000点リファレンス
Household Object Cross Structure Reference (20,000+ Points)

## 目的

家庭内にある一般的な物体をCross構造（20,000点以上）で表現し、JCrossコードで動かした際の6軸変化パターンを学習するための基礎資料。

カメラ学習時に「これがりんごのCross構造です」と照らし合わせながら見せるための視覚的リファレンス。

---

## Cross構造の基礎

### 6軸方向の意味

```
          UP (上)
           ↑
           |
LEFT ←─────●─────→ RIGHT
    (左)   |   (右)
           ↓
         DOWN (下)

    FRONT (前面・手前)
      ↗
     /
    ●
     \
      ↘
    BACK (背面・奥)
```

### 点密度と情報量

```
20,000点  = 基本的な形状認識
50,000点  = 詳細な特徴抽出
100,000点 = テクスチャまで認識
200,000点 = 超高精度（多層Layer 0相当）
```

## 1. 果物・食品カテゴリ

### 1-1. りんご（Apple）

**物理的特徴:**
- 球形に近い
- 直径: 約7-10cm
- 表面: 滑らか、光沢あり
- 色: 赤系が主（緑、黄色もあり）

**Cross構造（20,000点）:**

```
【空間配置】
- 中心から放射状に点が分布
- 球状の点群（ほぼ全方向均等）
- 中心付近の密度が高い

【6軸の値】
UP軸:    0.45-0.55 (上半球)
DOWN軸:  0.45-0.55 (下半球) ※ほぼ対称
RIGHT軸: 0.48-0.52 (右半球)
LEFT軸:  0.48-0.52 (左半球) ※ほぼ対称
FRONT軸: 0.60-0.80 (表面の赤色成分)
BACK軸:  0.20-0.40 (補色・青成分)
```

**JCrossコードでの表現:**

```jcross
# りんごのCross構造を生成
生成する 球体 = {
  "中心": [0, 0, 0],
  "半径": 0.4,
  "点数": 20000,
  "色": "赤"
}

# 横（RIGHT/LEFT軸）に回転
実行する 回転.横 = {
  "軸": "Y",
  "角度": 90,
  "対象": 球体
}

→ 結果: 球体なのでRIGHT/LEFT軸の値はほぼ変化なし（0.48-0.52を維持）
→ UP/DOWN軸も対称性により変化なし

# 縦（UP/DOWN軸）に回転
実行する 回転.縦 = {
  "軸": "X",
  "角度": 90,
  "対象": 球体
}

→ 結果: 同様にUP/DOWN軸も対称なので値は変化なし

# 前後（FRONT/BACK軸）に回転
実行する 回転.前後 = {
  "軸": "Z",
  "角度": 90,
  "対象": 球体
}

→ 結果: 表面色は均一なので、FRONT/BACKの色成分は変化なし
→ ただし光沢がある場合、角度により反射が変わる可能性
```

**動的変化パターン:**
- **回転**: どの軸でも見た目がほぼ同じ（球対称）
- **落下**: DOWN軸の値が時間とともに減少（重力）
- **転がり**: RIGHT/LEFT軸が周期的に変化

### 1-2. バナナ（Banana）

**物理的特徴:**
- 細長い湾曲形状
- 長さ: 約15-20cm
- 表面: 滑らか
- 色: 黄色

**Cross構造（20,000点）:**

```
【空間配置】
- 細長い曲線に沿って点が分布
- 湾曲方向に点が集中
- 非対称な分布

【6軸の値】
UP軸:    0.40-0.60 (湾曲の外側)
DOWN軸:  0.40-0.60 (湾曲の内側)
RIGHT軸: 0.20-0.80 (細長い方向に偏り)
LEFT軸:  0.20-0.80
FRONT軸: 0.70-0.90 (黄色の明るさ)
BACK軸:  0.10-0.30
```

**JCrossコードでの表現:**

```jcross
# バナナの湾曲形状
生成する バナナ = {
  "形状": "曲線柱",
  "長さ": 0.8,
  "太さ": 0.15,
  "湾曲": 30度,
  "点数": 20000,
  "色": "黄"
}

# 横（RIGHT軸）に回転
実行する 回転.横 = {"軸": "Y", "角度": 90, "対象": バナナ}

→ 結果:
  RIGHT軸: 0.50 → 0.40 (横から見ると細く見える)
  FRONT軸: 変化なし（色は同じ）
  UP軸: 0.50 → 0.60 (湾曲が上を向く)

# 縦（UP軸）に回転
実行する 回転.縦 = {"軸": "X", "角度": 90, "対象": バナナ}

→ 結果:
  UP軸: 0.50 → 0.70 (縦に立つ)
  DOWN軸: 0.50 → 0.30
  RIGHT軸: 大きく変化（湾曲方向が変わる）
```

**動的変化パターン:**
- **回転**: 湾曲により各軸の値が大きく変化
- **横に転がる**: RIGHT/LEFT軸が周期的に変動（楕円軌道）
- **落下**: 湾曲のため回転しながら落ちる

## 2. 日用品カテゴリ

### 2-1. マグカップ（Mug）

**物理的特徴:**
- 円筒形
- 高さ: 約10cm、直径: 約8cm
- 取っ手あり（非対称）
- 内部が空洞

**Cross構造（20,000点）:**

```
【空間配置】
- 円筒の外壁に点が集中
- 取っ手部分に追加の点群
- 内部は空（点が少ない）
- 非対称（取っ手があるため）

【6軸の値】
UP軸:    0.60-0.80 (上端が開いている)
DOWN軸:  0.40-0.50 (底面が平ら)
RIGHT軸: 0.30-0.70 (取っ手側に偏り)
LEFT軸:  0.30-0.70
FRONT軸: 0.50-0.70 (表面の色・模様)
BACK軸:  0.30-0.50
```

**JCrossコードでの表現:**

```jcross
# マグカップの構造
生成する カップ = {
  "形状": "円筒",
  "高さ": 0.5,
  "半径": 0.2,
  "厚さ": 0.02,
  "点数": 18000
}

生成する 取っ手 = {
  "形状": "円環の半分",
  "位置": [0.2, 0, 0],
  "点数": 2000
}

合成する マグカップ = {カップ, 取っ手}

# 横（RIGHT軸）に回転 - 取っ手が見える角度
実行する 回転.横 = {"軸": "Y", "角度": 90, "対象": マグカップ}

→ 結果:
  RIGHT軸: 0.50 → 0.70 (取っ手が右に突き出る)
  LEFT軸:  0.50 → 0.30
  FRONT軸: 変化なし（正面の模様が見える）

# 横に180度回転 - 取っ手が反対側
実行する 回転.横 = {"軸": "Y", "角度": 180, "対象": マグカップ}

→ 結果:
  RIGHT軸: 0.50 → 0.30 (取っ手が左に見える)
  LEFT軸:  0.50 → 0.70

# 上下逆さま
実行する 回転.縦 = {"軸": "X", "角度": 180, "対象": マグカップ}

→ 結果:
  UP軸:   0.70 → 0.40 (底が上に)
  DOWN軸: 0.40 → 0.70 (開口部が下に)
```

**動的変化パターン:**
- **回転**: 取っ手の位置により非対称な変化
- **横に置く**: UP/DOWN軸が均等化、RIGHT/LEFT軸が変化
- **転がる**: 取っ手があるため不規則な転がり

## 3. 家具カテゴリ

### 3-1. 椅子（Chair）

**物理的特徴:**
- 座面 + 背もたれ + 4本脚
- 高さ: 約80cm
- 空間的に複雑な構造
- 多くの空洞あり

**Cross構造（20,000点）:**

```
【空間配置】
- 座面に点が密集（平面）
- 背もたれに点が分布（垂直面）
- 4本の脚に線状に点が分布
- 空間的に立体的で点が分散

【6軸の値】
UP軸:    0.70-0.90 (背もたれ・座面上部)
DOWN軸:  0.10-0.30 (脚の底部)
RIGHT軸: 0.40-0.60 (左右対称)
LEFT軸:  0.40-0.60
FRONT軸: 0.50-0.70 (座る側)
BACK軸:  0.30-0.50 (背もたれ外側)
```

**JCrossコードでの表現:**

```jcross
# 椅子の構造
生成する 座面 = {
  "形状": "平面",
  "サイズ": [0.4, 0.4],
  "高さ": 0.4,
  "点数": 6000
}

生成する 背もたれ = {
  "形状": "平面",
  "サイズ": [0.4, 0.3],
  "位置": [0, 0.5, -0.2],
  "角度": 95度,
  "点数": 5000
}

生成する 脚 = {
  "形状": "円柱",
  "本数": 4,
  "高さ": 0.4,
  "点数": 9000
}

合成する 椅子 = {座面, 背もたれ, 脚}

# 横（RIGHT軸）に回転
実行する 回転.横 = {"軸": "Y", "角度": 90, "対象": 椅子}

→ 結果:
  FRONT軸: 0.60 → 0.50 (正面から側面に)
  RIGHT軸: 0.50 → 0.60 (側面が見える)
  UP軸: 変化なし（背もたれは上部のまま）

# 倒す
実行する 回転.縦 = {"軸": "X", "角度": 90, "対象": 椅子}

→ 結果:
  UP軸:    0.80 → 0.50 (背もたれが横になる)
  DOWN軸:  0.20 → 0.50
  FRONT軸: 0.60 → 0.80 (座面が前を向く)
```

**動的変化パターン:**
- **回転**: 複雑な形状のため6軸すべてが変化
- **倒れる**: UP/DOWN軸が急激に変化、最終的に均等化
- **引きずる**: 脚の接地点がRIGHT/LEFT方向に移動

- 開く角度により形状が変化
- 画面とキーボード
- サイズ: 約30×20×2cm

```
RIGHT軸: 0.45-0.55 (左右対称)
```

**JCrossコードでの表現:**

```jcross

  "サイズ": [0.6, 0.4],
  "厚さ": 0.01,
  "点数": 10000,
  "明るさ": 0.8

  "形状": "平面",

# 初期状態: 閉じている（0度）

  "終了角度": 120,
```

- **開閉**: UP軸とFRONT軸が連動して変化
- **持ち上げる**: 全体のZ座標が上昇

## 7. JCrossコード動的変化の一般パターン

### 7-1. 回転による6軸変化

```jcross
# 一般的な物体の回転

物体の形状タイプ別の変化:

【球体（りんご、ボールなど）】
回転 → ほぼ変化なし（全方向対称）
  UP/DOWN/RIGHT/LEFT軸: 0.45-0.55のまま

【円柱（ペン、ペットボトルなど）】
長軸を中心に回転 → 変化なし（回転対称）
短軸を中心に回転 → UP/DOWN軸が周期的に変化
  UP軸: 0.5 → 0.9 → 0.5 → 0.1 → 0.5 (360度で1周期)

【直方体（本、箱など）】
任意の軸で回転 → 大きく変化（非対称）
  90度回転でほぼ完全に軸の値が入れ替わる

【複雑形状（椅子、パソコンなど）】
回転 → 非線形な変化（予測困難）
  各軸の値が複雑に変化
```

### 7-2. 移動による6軸変化

```jcross
# 移動パターン

【平行移動（位置だけ変わる）】
実行する 移動 = {
  "方向": [1, 0, 0],  # RIGHT方向
  "距離": 2.0
}

→ 結果: 6軸の値は変化なし（形状は同じ）
→ ただし絶対座標は変化

【落下（重力による移動）】
実行する 落下 = {
  "初速度": 0,
  "重力": 9.8,
  "時間": 2.0
}

→ 結果:
  DOWN軸: 時間とともに値が減少（下に移動）
  他の軸: 変化なし

  時刻0秒: DOWN軸 = 0.5
  時刻1秒: DOWN軸 = 0.3
  時刻2秒: DOWN軸 = 0.1（地面に接触）

【転がる（回転しながら移動）】
実行する 転がる = {
  "方向": "RIGHT",
  "回転軸": "Z",
  "速度": 1.0
}

→ 結果:
  RIGHT軸: 時間とともに増加（移動）
  UP/DOWN軸: 周期的に変化（回転）
```

### 7-3. 変形による6軸変化

```jcross
# 変形パターン

【開く（ノートパソコン、本など）】
実行する 開く = {
  "ヒンジ位置": [0, 0, 0],
  "開始角度": 0,
  "終了角度": 120,
  "時間": 2.0
}

→ 結果:
  角度0度:   UP軸=0.50, FRONT軸=0.30
  角度60度:  UP軸=0.65, FRONT軸=0.70
  角度120度: UP軸=0.75, FRONT軸=0.85

  ※非線形な変化カーブ

【潰れる（柔らかい物体）】
実行する 圧縮 = {
  "方向": "DOWN",
  "力": 10.0,
  "時間": 1.0
}

→ 結果:
  UP軸:    0.70 → 0.40（高さ減少）
  DOWN軸:  0.30 → 0.60
  RIGHT/LEFT軸: 0.50 → 0.60（横に広がる）
```
