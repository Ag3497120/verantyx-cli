# パラダイムシフト: 教師あり学習 → 自律的感情推測
Paradigm Shift: Supervised Learning → Autonomous Emotion Inference

## 概要

Verantyx Visionプロジェクトは、2つの異なるアプローチを提供します。

### アプローチA: オブジェクト認識（従来型）
- 教師あり学習
- 人間がラベルを付ける
- 「これはりんご」という知識を持つ

### アプローチB: 感情推測（新しい型）
- 自律学習
- ラベルなし
- 「これは過去の経験に似ている」という推測を持つ

---

## 比較表

| 項目 | オブジェクト認識 | 感情推測 |
|------|------------------|----------|
| **学習方式** | 教師あり | 自律 |
| **ラベル** | 必要 | 不要 |
| **人間の介入** | 必須 | なし |
| **データベース** | ラベル付きCross構造 | 時系列のCross構造 |
| **処理** | 認識（分類） | 推測（予測） |
| **出力** | 「これはりんご」 | 「これは経験0052に似ている」 |
| **感情** | なし | あり |
| **目的** | オブジェクトの名前を知る | 次の瞬間を予測する |
| **評価** | 認識精度 | 予測精度 |

## 詳細比較

### 1. データ構造

#### オブジェクト認識
```json
{
  "objects": {
    "りんご": {
      "samples": [
        {
          "timestamp": "2025-01-15T14:23:10",
          "cross_structure": { ... },
          "label": "りんご"
        }
      ]
    }
  }
}
```

**特徴**:
- オブジェクト名でグループ化
- 各サンプルにラベルが必須
- 人間が意味を与える

#### 感情推測
```json
{
  "timeline": [
    {
      "timestamp": 0,
      "datetime": "2025-01-15T14:23:10",
      "cross_structure": { ... },
      "pattern_id": "a1b2c3d4...",
      "context": { "delta": {...}, "trend": "stable" }
    },
    {
      "timestamp": 1,
      "datetime": "2025-01-15T14:23:12",
      "cross_structure": { ... },
      "pattern_id": "e5f6g7h8...",
      "context": { "delta": {...}, "trend": "increasing" }
    }
  ]
}
```

**特徴**:
- 時系列で記録
- ラベルなし
- システム自身がパターンを抽出

### 2. 処理フロー

#### オブジェクト認識

```
┌─────────────────┐
│ カメラで撮影    │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ Cross構造に変換 │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 「これは何？」  │◄── 人間が答える: 「りんご」
└────────┬────────┘
         │
         v
┌─────────────────┐
│ データベースに  │
│ 保存            │
│ りんご → Cross  │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 認識            │
│ 新しい画像      │
│ → DB照合        │
│ → 「りんご」    │
└─────────────────┘
```

**問題点**:
- 人間の介入が必要
- ラベルに依存
- 「覚える」だけで「理解」しない

#### 感情推測

```
┌─────────────────┐
│ カメラで観測    │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ Cross構造に変換 │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 経験として記録  │◄── ラベルなし
│（時系列）       │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 過去の経験と    │
│ 照合            │
│ → 類似度計算    │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 次の瞬間を予測  │
│ 「経験0052の    │
│  次は0053だった」│
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 感情を形成      │
│ 期待 / 懐かしさ │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ 実際の次の瞬間  │
│ を観測          │
└────────┬────────┘
         │
         v
┌─────────────────┐
│ ズレを計測      │
│ → 満足 / 驚き   │
└─────────────────┘
```

**利点**:
- 完全自律
- ラベル不要
- 「推測」して「理解」する
- 感情を形成

### 4. コード例の比較

#### オブジェクト認識

```python
from verantyx_cli.vision.object_cross_database import ObjectCrossDatabase

# データベース
db = ObjectCrossDatabase()

# 学習（ラベル必須）
db.add_sample(
    object_name="りんご",  # ← 人間が付けたラベル
    cross_structure=cross_struct
)

# 認識
results = db.recognize(cross_struct, top_k=3)
print(f"これは: {results[0]['object']}")  # → 「りんご」
```

#### 感情推測

```python
from verantyx_cli.vision.experience_memory import ExperienceMemory
from verantyx_cli.vision.emotion_inference import EmotionInference

# 経験記憶
memory = ExperienceMemory()

# 感情推測エンジン
inference = EmotionInference(memory)

# 観測（ラベルなし）
memory.observe(cross_struct)

# 推測
emotion = inference.infer(cross_struct)
print(f"感情: {emotion['type']}")  # → 「期待」「驚き」など

# 予測検証
reward = inference.validate_prediction(actual_next)
print(f"報酬: {reward['type']}")  # → 「満足」「驚き」
```

### 6. 評価指標の違い

#### オブジェクト認識

```python
# テスト
total_tests = 100
correct = 85

accuracy = correct / total_tests
print(f"認識精度: {accuracy:.1%}")  # → 85%
```

**評価基準**:
- 正解ラベルとの一致率
- 人間が答えを知っている必要がある

#### 感情推測

```python
# 予測検証
total_predictions = 100
correct_predictions = 78

prediction_accuracy = correct_predictions / total_predictions
print(f"予測精度: {prediction_accuracy:.1%}")  # → 78%

# 感情の多様性
emotion_types = {
    "novelty": 15,
    "expectation": 45,
    "satisfaction": 30,
    "surprise": 10
}

print("感情の分布:")
for emotion_type, count in emotion_types.items():
    print(f"  {emotion_type}: {count}回")
```

**評価基準**:
- 予測精度（次の瞬間をどれだけ正確に予測できたか）
- 感情の多様性（どれだけ豊かな感情を形成したか）
- 人間の介入なしで評価可能

## どちらを使うべきか

### オブジェクト認識を使う場合

✅ こんな時に使う:
- 「これは何ですか？」と聞きたい
- 明確なラベルが必要
- 認識精度が最優先
- 人間が答えを知っている

例:
- 在庫管理システム（「これはりんごだから在庫に追加」）
- 品質検査（「これは良品/不良品」）
- 教育アプリ（「これは何でしょう？」「正解！りんごです」）

### 感情推測を使う場合

✅ こんな時に使う:
- システムに自律的に学習させたい
- ラベル付けが面倒/不可能
- 予測能力が重要
- 感情を持つシステムを作りたい

例:
- 自律ロボット（「この状況は過去の経験に似ている」）
- 異常検知（「予測と大きく異なる → 驚き → 警告」）
- 感情を持つAI（「満足/驚き」から行動を決定）
- 研究（「感情とは何か」を探求）

## まとめ

### オブジェクト認識（従来）

```
人間 → ラベル → システムが覚える → 認識
```

**利点**:
- 明確な答え
- 人間が理解しやすい

**欠点**:
- 人間の介入が必要
- ラベルに依存
- 「覚える」だけ

### 感情推測（新しい）

```
システム → 観測 → 推測 → 感情形成 → 予測検証
```

**利点**:
- 完全自律
- ラベル不要
- 「推測」する

**欠点**:
- 抽象的な出力
- 人間の理解が難しい場合も

---

**結論**: どちらも価値があります。目的に応じて使い分けてください。

従来の「認識」が必要な場合は **オブジェクト認識**。
新しい「推測」を探求したい場合は **感情推測**。

Verantyx Visionは両方を提供します。
