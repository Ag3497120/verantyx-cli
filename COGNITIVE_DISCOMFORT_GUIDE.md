# 認知的不快感システム
Cognitive Discomfort System

---

## 🎯 概要

**「分からない」「予測が外れた」は不快**

従来の生理的不快感（体温、エネルギー、痛み、酸素）だけでなく、**認知的不快感**を実装しました。

---

## 😣 認知的不快感の種類

### **1. 予測誤差不快（Prediction Error Discomfort）**

```
予測が外れる → 不快
予測が当たる → 快
```

**計算式:**
```python
if prediction_error < 0.05:
    return 0.0  # ほぼ予測通り → 不快なし

if prediction_error > 0.5:
    return 1.0  # 大きく外れた → 最大不快

discomfort = prediction_error * 2.0  # 線形に増加
```

**例:**
- 予測誤差 0.02 → 不快感 0.0（スッキリ）
- 予測誤差 0.25 → 不快感 0.5（モヤモヤ）
- 予測誤差 0.6 → 不快感 1.0（気持ち悪い）

---

### **2. 未知性不快（Novelty Discomfort）**

```
未知のもの → 不快（不安）
知っているもの → 快（安心）
```

**計算式:**
```python
if similar_experiences_count == 0:
    return 1.0  # 全く未知 → 最大不快

if similar_experiences_count >= 10:
    return 0.0  # 十分な経験 → 不快なし

discomfort = 1.0 - (similar_experiences_count / 10.0)
```

**例:**
- 類似経験 0個 → 不快感 1.0（怖い）
- 類似経験 5個 → 不快感 0.5（ちょっと不安）
- 類似経験 10個 → 不快感 0.0（安心）

---

### **3. 複雑性不快（Complexity Discomfort）**

```
複雑すぎる → 理解できない → 不快
シンプル → 理解できる → 快
```

**計算式:**
```python
if total_points < 3000:
    return 0.0  # シンプル → 不快なし

if total_points > 10000:
    return 1.0  # 超複雑 → 最大不快

discomfort = (total_points - 3000) / 7000.0
```

**例:**
- 2,500点 → 不快感 0.0（シンプル）
- 6,500点 → 不快感 0.5（ちょっと複雑）
- 12,000点 → 不快感 1.0（複雑すぎる）

---

### **4. 同調失敗不快（Synchronization Failure Discomfort）**

```
既存パターンに当てはまらない → 不快
パターンに当てはまる → 快
```

**計算式:**
```python
if sync_failure < 0.3:
    return 0.0  # ある程度同調 → 不快なし

if sync_failure > 0.8:
    return 1.0  # 全く同調できない → 最大不快

discomfort = (sync_failure - 0.3) / 0.5
```

---

## 😊 認知的快感（報酬）

### **1. 予測成功快（Prediction Success Pleasure）**

```
予測が当たった → 快感
```

**計算式:**
```python
if prediction_error < 0.05:
    return 1.0  # バッチリ当たった！

if prediction_error > 0.3:
    return 0.0  # 外れた → 快感なし

pleasure = 1.0 - (prediction_error / 0.3)
```

---

### **2. 理解快（Understanding Pleasure）**

```
理解できた → 快感
```

**計算式:**
```python
if similar_experiences_count >= 5:
    return 1.0  # よく知ってる！

if similar_experiences_count == 0:
    return 0.0  # 未知 → 快感なし

pleasure = similar_experiences_count / 5.0
```

---

### **3. 同調快（Synchronization Pleasure）**

```
パターンに当てはまった → 快感
```

**計算式:**
```python
if sync_degree > 0.8:
    return 1.0  # ピッタリ！

if sync_degree < 0.3:
    return 0.0  # 当てはまらない

pleasure = (sync_degree - 0.3) / 0.5
```

## 🎚️ 総合不快感の計算

### **認知的不快感**

```python
total_cognitive_discomfort = (
    prediction_error_discomfort * 0.4 +  # 予測誤差が最も重要
    novelty_discomfort * 0.3 +           # 未知性も重要
    complexity_discomfort * 0.2 +        # 複雑性
    sync_failure_discomfort * 0.1        # 同調失敗
)
```

### **総合不快感（生理的 + 認知的）**

発達段階に応じて重み付けが変化:

#### **0歳_新生児**
```python
total = (physiological * 0.3) + (cognitive * 0.7)
# 認知的不快が支配的（理解できないことが多い）
```

#### **1歳_立位**
```python
total = (physiological * 0.4) + (cognitive * 0.6)
# まだ認知的不快が強い
```

#### **3歳_歩行**
```python
total = (physiological * 0.5) + (cognitive * 0.5)
# バランス
```

#### **7歳以降**
```python
total = (physiological * 0.6) + (cognitive * 0.4)
# 生理的不快がより重要に
```

## 🎯 不快感解消アクション

最も不快な要素に応じて、解消アクションを決定:

### **予測誤差が最大**
```
アクション: 予測モデルを更新
理由: 予測が外れて不快
```

### **未知性が最大**
```
アクション: 類似経験を探す
理由: 未知で不安
```

### **複雑性が最大**
```
アクション: 単純化・分解
理由: 複雑すぎて理解できない
```

### **同調失敗が最大**
```
アクション: 新しいパターンとして記憶
理由: 既存パターンに当てはまらない
```

## 💡 重要な洞察

### **1. 未知は不快**

初めて見るものは不快（不安）。これが**好奇心**の原動力。

```
未知 → 不快 → 理解したい → 探索 → 理解 → 快感
```

### **2. 予測が外れると不快**

予測誤差が大きいと不快。これが**学習**の動機。

```
予測失敗 → 不快 → モデル更新 → 予測成功 → 快感
```

### **3. 理解は快感**

既存パターンに当てはまると快感。これが**報酬**。

```
未知 → 探索 → 理解 → 同調成功 → 快感（報酬）
```

### **4. 発達段階による変化**

- **0歳**: 認知的不快が支配的（何も分からない）
- **18歳**: 生理的不快が重要（理解は当たり前）
