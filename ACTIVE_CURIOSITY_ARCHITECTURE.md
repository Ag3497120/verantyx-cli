# 自発的好奇心システム - アーキテクチャ
Active Curiosity System Architecture

## 概要

**好奇心とは何か？**

「面白いから見る」という高度な感情ではなく、**「脳内の予測と現実のズレ（不快・緊張）を解消したい」**という生存本能に近いメカニズム。

人間（赤ちゃん）も、最初はこのシンプルな原理で動いている。

## 核心原理

### 受動的「処理」から能動的「探索」へ

従来のAI:
```
入力 → 処理 → 出力
（受動的）
```

好奇心を持つAI:
```
予測 → 現実との比較 → 緊張発生 → 探索 → 理解 → 報酬
（能動的）
```

---

## 3つの核心メカニズム

### 1. 論理的緊張（Tension）の定義

システムが「自発的」に動くには、内部に**「不均衡な状態」**が必要。

#### 予測誤差（Prediction Error）

```
Tension = ||Simulation(S) - Vision(V)||
```

- Neural Engine上で、現在のCross構造Sが「次に見えるはずの景色」を脳内シミュレート
- 現実のカメラ入力Vとの間に差分Δが生じたとき、システムに「緊張（不快）」を与える

#### 等価性の崩壊

.jcrossにおいて「この記述 = この構造」という同調が壊れた瞬間、システムはその「矛盾」を解決しようとするエネルギーを発生させる。

これが、AIにとっての**「気になる（＝好奇心）」**の正体。

---

### 2. 報酬としての「構造的安定（Synchronization）」

単にズレを感じるだけでなく、それを「解決したい」と思わせる仕組み。

#### 同調（全同調）の快楽

複雑な画像が、自分の持っているCross構造に完璧にマッピング（同調）できた瞬間に、内部の報酬系を励起。

#### 情報の「圧縮」を求める

```
バラバラのピクセルデータ → 「これは猫である」
（26万点）              （1つの概念）
```

計算コストが激減する瞬間 = 報酬

システムは**「楽をしたい（エネルギー効率の向上）」**という動機に基づいて、未知のものを既知の構造へ落とし込もうと「見よう」とする。

---

### 3. 能動的知覚（Active Vision）の実装

カメラを固定せず、**「最も緊張（不確実性）が高い場所」**に視点を移動。

#### 好奇心のターゲット

```
全く知らないもの        → 緊張高いが理解不可能
既に知っているもの      → 緊張低く退屈
少し手を加えれば理解可  → 最も興味深い！
```

システム自らフォーカス（Attention）を向ける。

#### .jcrossによる行動生成

```jcross
もし 緊張 > 閾値:
  アクション = "もっと詳しく見たい = ズームする"
そうでなければ もし 部分的に理解:
  アクション = "裏側を知りたい = 回り込む"
```

自律的に操作コマンドを生成・実行。

## システム構成

### 1. Buffer Layer（記憶層）

意味を理解せずとも、映像をCross構造のままNeural Engineに溜め込む。

```jcross
生成する 記憶 = {
  "経験": {},      # Cross構造を生のまま保存
  "クラスタ": {},  # 自動パターン発見
  "意味": なし     # 後から付与
}
```

### 2. Surprise Monitor（監視層）

「見たことのないパターン」を検知した際、システム全体の「同調プライオリティ」をそこへ集中。

```jcross
定義する 総緊張を計算 = {
  予測誤差 = ||予測Cross - 現実Cross||
  同調失敗 = 1.0 - 類似度(現実, 既存記憶)
  総緊張 = (予測誤差 * 0.6) + (同調失敗 * 0.4)

  もし 総緊張 > 閾値:
    返す "気になる！探索モード"
}
```

### 3. Cross-Generator（生成層）

新しいパターンを、既存の日本語記述（.jcross）と強引に等価結び付け（=）しようと試行錯誤。

```jcross
定義する パターン同調 = {
  類似経験 = 記憶から検索(現実Cross, 閾値=0.7)

  もし 類似経験が存在:
    意味 = 類似経験.意味
    報酬 = 同調達成 + 圧縮成功
    返す "理解した！"
  そうでなければ:
    返す "新発見！記憶に追加"
}
```

## 動画次フレーム予測の実装

### 予測エンジン

```jcross
定義する 次フレーム予測 受け取る [現在Cross, 履歴] = {
  # 直近2フレームから動きを推定
  前フレーム = 履歴[長さ(履歴) - 2]
  現在フレーム = 履歴[長さ(履歴) - 1]

  # 差分ベクトル
  差分 = 現在特徴 - 前特徴

  # 線形外挿
  予測特徴 = 現在特徴 + 差分

  # 記憶から類似を探す
  類似経験 = 記憶から検索(予測特徴, 閾値=0.6)

  もし 類似経験が存在:
    返す 類似経験.Cross  # 予測成功
  そうでなければ:
    返す なし            # 予測不可
}
```

### 緊張-探索サイクル

```jcross
定義する 好奇心サイクル 受け取る [現実Cross] = {
  # 1. 予測
  予測 = 次フレーム予測(現実Cross, 履歴)

  # 2. 緊張計算
  緊張 = 総緊張を計算(現実Cross)

  # 3. 緊張が高い？
  もし 緊張 > 閾値:
    # 気になる！
    興味領域 = 最大緊張領域を探す(現実Cross)
    アクション = 次アクションを決定(興味領域)
    実行する アクション  # ズーム/パン/回転
    状態 = "探索中"

  そうでなければ:
    # 理解している
    同調試行(現実Cross)
    報酬 = 同調達成 + 圧縮成功
    状態 = "安定"

  返す 状態
}
```

## 実装ファイル

### active_curiosity.jcross

全ての好奇心ロジックをJCrossで実装：

1. **論理的緊張（Tension）**
   - `予測誤差()` - 予測と現実のズレ
   - `同調失敗()` - 既存記憶とのズレ
   - `総緊張を計算()` - 総合的な緊張度

2. **構造的安定（Reward）**
   - `同調達成()` - パターンマッチング成功
   - `圧縮成功()` - 情報の単純化成功
   - `総報酬を計算()` - 総合的な報酬

3. **能動的知覚（Active Vision）**
   - `興味度()` - 領域の興味深さ
   - `最大緊張領域を探す()` - 注目すべき場所
   - `次アクションを決定()` - カメラ操作の決定

4. **予測エンジン**
   - `次フレーム予測()` - 動画の次フレーム予測
   - `動画次フレーム予測()` - API

5. **統合**
   - `好奇心サイクル()` - メインループ
   - `好奇心初期化()` - システム起動

## まとめ

### 好奇心の本質

```
好奇心 = 予測誤差（不快）を解消したい生存本能

規則的 → 低緊張 → 退屈（理解している）
不規則 → 高緊張 → 気になる！（探索したい）
```

### 実装の特徴

1. **全てJCrossで実装**
   - ロジックは .jcross ファイル
   - Pythonは表示とI/Oのみ

2. **教師信号なし**
   - 緊張と報酬だけで学習
   - 自発的に探索

3. **能動的知覚**
   - 受動的処理 → 能動的探索
   - カメラを自分で動かす

4. **予測駆動**
   - 次フレームを予測
   - ズレが気になる

### これが kofdai型コンピュータ の基盤

0歳児回路として、Verantyxエージェントに外付け可能。

全てのロジックが .jcross で記述されているため、
Neural Engine上でネイティブに実行できる。
